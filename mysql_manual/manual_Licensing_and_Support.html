3  MySQL的使用授權和技術支援

本章描述 MySQL 使用授權和技術支援安排，包括：

  • 我們對於非微軟和微軟作業系統使用授權的政策﹔
  • MySQL 分發遵從的版權(見3.2  MySQL 使用的版權)﹔
  • 當申請一個使用授權時，演示一個例子(見3.4 使用授權實例）﹔
  • 使用授權和支援費用(見3.5  MySQL 的使用授權和技術支援費用)，及技術支援的好處(見 3.6 商業性支援的類型）﹔

3.1  MySQL的使用授權政策

針對非微軟作業系統如Unix或 OS/2 的使用授權的正式條款在 J 針對非微軟作業系統的 MySQL 伺服器使用授權中說明。基本上，我們使用授權政策如下：

  • 對於一般的內部使用，MySQL通常是免費的。如果你不想，就不必付錢給我們。
  • 一個使用授權是必需的，如果：
    • 你直接銷售MySQL伺服器或作為其他產品或服務的一部分﹔
    • 你在某些客戶那裡為了安裝和維護一個 MySQL 伺服器而收費﹔
    • 你在不可再分發的分發中包括 MySQL並且你對該分發的某些部分收費﹔
  • 在必須有 MySQL 使用授權的情況下，對每台運行mysqld伺服器的機器，你都需要一個使用授權，但多CPU機器按單CPU計算，並且在一台機器上運行 MySQL 伺服器的數量或並發連接到這台運行一個伺服器的機器的客戶數量也無限制！
  • 你在商業程式使用包含客戶代碼不需要一個使用授權，MySQL的客戶端存取部分不屬公共領域，mysql命令行客戶程式包含在 GNU 通用使用授權下的 readline 庫的代碼。
  • 對於已經購買了 10 個使用授權或一種足夠級別的技術支援的消費者，我們提供附加的功能。目前，這意味著我們提供 myisampack實用工具，它能產生快速的壓縮的唯讀資料庫（伺服器支援讀取這樣的資料庫，但不包含用於產生它們的壓縮工具)。當支援協議產生了足夠的收入時，我們將在與MySQL 伺服器同樣的使用授權下發行這個工具。
  • 如果你使用不需要一個使用授權的 MySQL，但是你的確喜歡 MySQL 並且有志於更進一步的開發，無論如何肯定歡迎你購買一個使用授權。
  • 如果你在一個商業環境中使用 MySQL以便通過它獲利，我們要求你購買一定級別的技術支援以推進開發。我們感到，如果 MySQL 有助於你的業務，要求你幫助MySQL也是合理的（否則，如果向我們你詢問支援問題，你不僅是正在免費使用我們傾注大量精力的產品，而且你正在要求我們提供免費的支援。）。

對於在微軟作業系統下面( Win95/Win98/WinNT)的使用，你在一個 30 天的試用期後需要一個 MySQL 使用授權，除了教育用途或大學或政府資助的研究機構的使用授權可免費申請獲得，見 K 針對微軟作業系統的 MySQL 使用授權。一個共享軟件版本的 MySQL -Win32 可在購買前從 http://www.mysql.com/mysql_w32.htmy 下載試用。在你付錢後，你將得到一個密碼讓你能夠訪問最新 MySQL - Win32 版本。

如果你對你特定的 MySQL使用是否需要一個使用授權有任何問題，請聯繫我們，見 3.5.2 聯繫資訊.

如果你需要一個 MySQL 使用授權，最容易的付款方法是使用在 https://www.mysql.com/license.htmy網站的 Tcx 的安全伺服器上的使用授權表格。其他付費形式在3.5.1 付款資訊討論。

3.2 MySQL使用的版權

MySQL 分發有若干不同的版權：

  1. 構造mysqlclient庫和“client”目錄下的程式所需的MySQL特定原始程式使屬於公共領域(public domain)，屬於公共領域的每個文件有一個文件頭清楚地說明這點。這包括在“client”目錄下的所有東西和mysys、mystring和dbug庫的某些部分。
  2. 原始程式的某一小部分( GNU  getopt)遵從“ GNU 庫文件通用使用授權(GLPL)”，參見"mysys/COPYING.LIB"文件。
  3. 原代碼的某一小部分( GNU  readline)遵從“ GNU 公共使用授權”。見"readline/COPYING"文件。
  4. 原始程式的某些部分(regexp 庫)遵從Berkerley風格的版權。
  5. 在非微軟平台上的MySQL伺服器所需的其他原始程式遵從“ MySQL 自由公共使用授權”，它基於“阿拉丁自由公共使用授權”。見J 針對非微軟作業系統的 MySQL 伺服器使用授權。當在任何微軟作業系統上運行MySQL時，使用另外的使用授權。

下列幾點闡明了我們的版權政策後的理念：

  • SQL客戶庫應該是完全免費的，以使它能不受限制地包含在商業產品中。
  • 想要自由存取我們花了很多精力開發的軟件的任何人都可以擁有它，只要他們不試圖借助分發它以盈利為目而直接賺錢。
  • 想要有權保護他們自己的軟件專利，但也從我們的工作得到價值的任何人，可以為此權力付費。
  • 這意味著正常的家庭使用是免費的，但是如果你為一些對你很重要的事使用 MySQL，你可以通過購買一個使用授權或一個技術支援合同來幫助進一步開發它。見3.6 商業性支援的類型。

 3.2.1 可能的未來版權改變

我們可能在將來用GPL分發較老的 MySQL 版本，然而，這些版本將被標明為 GNU  MySQL。另外，在相關文件的所有版權通告將改為 GPL 。

3.3 MySQL商業性分發

本節澄清“ MySQL 自由公共使用授權”( FPL )中提出的使用授權條款。見 J 針對非微軟作業系統的 MySQL 伺服器使用授權。

 MySQL可以是自由地使用，包括被商業實體的評估或不受支援的內部使用。然而，MySQL商業目的分發，或任何部分或全部包含或從MySQL派生的東西，都需要從 TcX  AB 申請一個書面的商業使用授權，它是唯一的授權發証實體。

你不能“免費”地在一個含有任何收費的東西的軟件包括含MySQL，但下面所說的除外。

在使用授權的第二款提供了例外的目的是允許運營一個FTP伺服器或公告牌的商業組織免費分發 MySQL，如果：

  1. 該組織遵守 FPL 的其他條款，其包括要求分發 MySQL 和任何派生產品的全部原始程式，並且與MySQL一起分發FPL本身﹔
  2. 為下載 MySQL 的唯一費用是基於分發服務的費用，而不是基於正在被檢索的資訊內容(即，費用應該與檢索同樣大小的隨機位組合是一樣的)﹔
  3. 伺服器或BBS可由一般公眾訪問，即，電話號碼或IP地址沒被保密，且任何人可以獲得資訊的存取(可能通過交訂費或存取費用，該費用不依賴於或相關於購買其他東西)。

如果你想要以商業形式分發集成了MySQL的軟件，並且你不想滿足這些條件，你應該聯繫 TcX  AB 尋求商業使用授權，它涉及付費。你能合法地分發 MySQL 或任何包含 MySQL的東西的唯一方法是在FPL的要求下分發 MySQL，或通過從 TcX  AB 獲得一個商業使用授權。

3.4 使用授權實例

本節描述一些實例以闡述你是否許可使用 MySQL，通常這些例子包括提供 MySQL 伺服器作為一個產品的一部分，或你正在向客戶銷售服務，或要求 MySQL與你的產品一起使用。在這些的情況中，如果必要，為客戶申請一個使用授權是你的責任。（如果你的客戶已經有了 MySQL 使用授權，這個要求就不考慮了，但是賣方必須將客戶資訊和和使用授權號傳到TcX，並且使用授權必須是一個完整的使用授權，而不是一個OEM使用授權。）

注意單個 MySQL 使用授權涵蓋了一台機器上的任何數量的CPU/用戶/客戶的mysqld的伺服器！

3.4.1 銷售使用MySQL的產品

要確定在銷售你的應用程式時，你是否需要一個 MySQL 使用授權，你應該詢問，應用程式的正確運行是否可能使用 MySQL ，且你是否在你的產品刈荺括 MySQL 。有幾種情況要考慮：

  • 你的應用是否需要 MySQL 才能正確工作？如果你的產品需要 MySQL，對於任何運行mysqld伺服器的機器，你都需要一個使用授權。例如，如果你已經圍繞MySQL設計了應用，然後你確實開發了需要該引擎的一個商業產品，因此你需要一個使用授權。如果你的應用不需要 MySQL，你不必獲得一個使用授權。例如，如果 MySQL 只是為你的產品加入一些新的可選功能(例如，如果 MySQL被用來增加日誌到資料庫中，而不是記錄到一個文本文件中)，它應該在正常使用範圍內，並且不需要一個使用授權。換句話說，如果你銷售為MySQL 而專門設計的或需要MySQL才能運行的產品，你需要一個使用授權，不管你是否是為你的客戶提供 MySQL來作為你產品的一部分，這也取決於你在為你的客戶做什麼。你計劃為你的客戶提供與軟件一起安裝 MySQL詳細的指令嗎？那麼你的產品可能使用 MySQL，如果是，你需要購買一個使用授權。如果你簡單地連接在購買你的軟件時你希望已經安裝資料庫的話，那麼你可能不需要一個使用授權。
  • 你在一個分發包括 MySQL並對此分發收費嗎？如果你在銷售給客戶的分發刈荺括 MySQL ，對每台運行mysqld伺服器的機器，你都將需要一個使用授權，因為在這種情況下，你正在銷售一個包括 MySQL的系統，不管與你的產品一起使用 MySQL是你必需的或可選的。
  • 你的產品既不需要也不包含 MySQL 嗎？假定你想銷售一個產品，它設計成通用的以使用“某些資料庫”並且能被配置成可使用它支援的多個其他資料庫選擇( MySQL， PostgreSQL，或其他)，即，你的產品不需要 MySQL，但是能支援任何基本功能水平的資料庫，且你不依賴任何只有 MySQL 才支援的東西。如果你的客戶實際上確實選擇使用MySQL，你們中有人欠我們錢嗎？在這種情況下，如果你不提供，為客戶獲得或設置 MySQL 應由客戶決定是否使用它，你也不需要一個使用授權。如果你確實施行那種服務，見3.4.2 銷售MySQL 相關的服務。

3.4.2 銷售MySQL相關的服務

如果你在一個客戶機器上安裝 MySQL 並為此服務收費(直接或間接地)，那麼你必須購買一個 MySQL 使用授權。

如果你銷售一個應用，而 MySQL 不是完全需要但是能被使用，可能要出示一個使用授權，這取決於怎樣建立 MySQL。假如你的產品既不需要 MySQL 也不在你的產品分發包含它，但是可以為有這種願望的那些客戶配置成使用 MySQL。（例如，你的產品可以使用任何資料庫引擎的一個，可能就是這種情況。）

如果客戶得到並且安裝 MySQL，不需要使用授權。如果你為你的客戶施行這種服務，那麼就需要一個使用授權，因為你正在銷售一種包含 MySQL 的服務。

3.4.3  ISP  MySQL 服務

網際網路服務供應商(ISP)經常為他們的客戶配備 MySQL 伺服器。

如果你是一個ISP，允許客戶自己在沒有你幫助的情況下安裝並且管理你機器上的MySQL，你和你的客戶都不需要一個MySQL 使用授權。

如果你對作為你客戶服務一部分的 MySQL安裝和管理的支援收費，那麼因為你正在銷售一種包括 MySQL服務，你需要一個使用授權。

3.4.4 運營一個使用 MySQL的Web伺服器

如果你與一個Web伺服器一起使用 MySQL，你不必支付一個使用授權費用。

就算你運營使用 MySQL的一個商業Web伺服器，也是這樣的，因為你不在銷售 MySQL 本身。然而，在這種情況下，我們希望你購買 MySQL 技術支援，因為 MySQL 正在幫助你的企業。

3.6  商業性支援的類型

3.6.1 基本的電子郵件支援

基本的電子郵件支援是很便宜的支援選擇，並且更應該認為它是作為支援我們開發 MySQL 的一種方式而不是真正的支援選擇。

在這支援級別上，MySQL 郵件列表是通信的最佳方法，問題通常應該寄給主郵件列表( [email protected] )或其它常規列表之一(例如對於有關Windows  MySQL 問題，[email protected] )，因為其他人可能已經經歷了並且解決了你的問題。見2.2 提問或報告錯誤。

然而，通過購買基本電子郵件支援，你也能訪問支援地址 [email protected]，它不作為你購買一個使用授權而得到很小支援的一部分，這意味著對於特別緊急的問題，你可以交叉郵寄你的消息到 [email protected]。(如果消息包含敏感數據，你應該只郵寄到 [email protected])。

記住！當你發送一條消息到時，總要包括你的登記號碼和過期日期。

基本電子郵件支援包括下列服務類型：
